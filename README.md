# 印税OS v3.0 Core Specification

**kazene-royalty-os-v3** is an open specification for a royalty-processing OS that converts AI access to creative works into a governed cycle of **permission, trace, and allocation**.

印税OS v3.0 は、AIによる著作物アクセスを  
**許諾・ログ・分配** の正規循環へ変換するための権利処理OS仕様です。

本仕様は、Qコインを **非換金・非譲渡・内部会計単位** として定義し、  
AIを利用者または契約主体の **代理エージェント** として扱うことで、  
透明性・権利者保護・対価還元を両立する最小構造を提示します。

---

## 概要

生成AI時代における著作物利用は、しばしば次の二極に分かれがちです。

- 無断利用・不透明利用
- 厳格な遮断・全面禁止

印税OS v3.0 は、その中間にある  
**「許諾されたアクセスを、ログと分配によって正規循環へ変換する」**  
という第三の道を設計します。

このリポジトリは、印税OSの思想・最小仕様・機械可読な骨格を公開し、  
将来的な実装・共同研究・制度設計・事業連携のための参照点となることを目的としています。

## 目的

印税OS v3.0 の目的は、主に次の4つです。

1. **AIによる著作物アクセスを可視化すること**  
   誰が、何を、どの範囲で利用したかをログとして記録する。

2. **権利者への還元ルートを設計すること**  
   AIアクセスが著者・出版社・権利者への対価還元に接続される構造をつくる。

3. **AI利用を無秩序な収奪ではなく、管理可能な循環へ変えること**  
   利用を止めるのではなく、整理し、許諾し、分配する。

4. **将来の標準的な権利処理基盤の原型を提示すること**  
   実装者・法務・出版社・行政・研究者が共有できる最小仕様を示す。

## 基本思想

印税OSは、コイン中心の構想ではありません。  
主役はあくまで **権利処理OS** です。

Qコインは、投機対象でも決済手段でもなく、  
AIアクセスの価値を測定し、分配を計算するための **内部会計単位** です。

この仕様は、次の発想に基づいています。

- AIは独立した読者ではなく、利用者または契約主体の代理として動く
- 著作物利用は、許諾・ログ・分配の3点が揃って初めて正規循環になる
- 透明性は、権利者保護とAI利用促進の両方に必要である
- 最初から巨大実装を目指すのではなく、最小循環を先に成立させるべきである

## コア原則

### 1. 代理原則
AIは独立した権利主体ではなく、  
**利用者または契約主体の代理エージェント** として著作物へアクセスします。

### 2. 非金融原則
Qコインは **通貨・暗号資産・投資商品ではありません。**  
非換金・非譲渡・外部送金不可の内部会計単位として扱います。

### 3. 透明性原則
すべてのアクセスは、  
**監査可能なログ** として保存されるべきです。

### 4. 還元原則
AI利用は、権利者への利益還元へ接続されるべきです。  
利用だけが流れ、還元が戻らない構造は不完全です。

### 5. 最小循環原則
v3.0 は完成系ではなく、  
**現実に動かせる最小構造** を優先します。

## 3レイヤー構造

印税OS v3.0 は、次の3レイヤーで構成されます。

### Access Layer
著作物ごとに、AIアクセスの権限を定義する層です。

例:

- 閲覧可
- 要約可
- 引用可
- 埋め込み可
- 外部応答利用可
- 再学習不可
- 商用利用可否
- 参照専用

ここでは、AIが何をできるかを **事前に定義** します。

---

### Trace Layer
AIが行ったアクセス行為を記録する層です。

例:

- どのAIが
- どの契約主体の代理として
- どの作品に
- いつ
- どんな行為を行い
- どれだけ参照し
- どの権限のもとで利用したか

このログは、単なる記録ではなく  
**分配の根拠** になります。

---

### Royalty Layer
アクセスログをもとに、内部会計単位 Qコインを計算し、  
著者・出版社・権利者への分配に変換する層です。

ここで重要なのは、Qコインを外へ出さないことです。

- 非換金
- 非譲渡
- 外部ウォレット不可
- 法定通貨交換不可
- 相場形成不可

印税OSは、金融商品を作るのではなく、  
**権利処理と分配計算のためのOS** を作ります。

---

## Qコインとは何か

Qコインは、印税OS内部でのみ用いられる  
**アクセス価値の計測単位** です。

### Qコインの定義
- 非換金
- 非譲渡
- 外部送金不可
- 外部売買不可
- 内部精算専用
- 分配計算用メトリクス

### Qコインの役割
- AIアクセス量の測定
- 行為別の負荷・価値の差の計測
- 作品ごとの分配基準の形成
- 利用上限や優先アクセス制御への応用
- 将来的なレポーティング基盤の標準化

Qコインは、経済圏の通貨というより  
**情報圏の価値指標** です。

## 分配の基本ロジック

v3.0 では、複雑すぎる設計は避け、  
納得可能な最小ロジックを採用します。

### 基本式
```text
Q消費量 = 基本単価 × 行為係数 × 作品係数 × 利用範囲係数
行為係数の例

view = 1.0

summarize = 1.5

quote = 1.2

external_response = 2.0

月次分配の例
月間分配額 = 総売上 × (当該作品のQ消費量 / 全作品のQ消費量)

v3.0 の目的は、完璧な価格理論ではなく、
利用 → 可視化 → 還元 の循環を成立させることです。

登場主体
著者

作品登録

利用条件設定

分配受領

作品管理

契約管理

分配調整

複数作品の一括運用

利用者

AI利用の主体

契約主体

AIへのアクセス指示者

AIエージェント

利用者の代理として作品へアクセス

要約・参照・引用補助を実行

単独では権限主体にならない

v3.0 が目指すもの

印税OS v3.0 は、次のような未来の入口を目指しています。

AIが作品にアクセスしても、権利者に還元が起こる

出版社が安心してAI許諾設計を導入できる

利用者が正規ルートでAIに読ませられる

実装者がログ・分配・権限の共通骨格を参照できる

行政・法務・研究者が議論の土台を共有できる

これは、AI利用を止めるための仕様ではありません。
AI利用を、正規化し、循環化し、文明化するための仕様 です。

安全線

印税OS v3.0 における安全線は、次の通りです。

許諾済み作品のみアクセス

AIは代理エージェントとして利用

全アクセスログを保存

再学習可否を作品ごとに定義

Qコインは内部専用

分配は円建てまたは内部残高で清算

ブロックチェーンを必須としない

最初は台帳型実装で十分とする

危険線

次のような設計は、v3.0 の範囲外です。

Qコインの外部売買

外部ウォレット対応

ユーザー間譲渡

無許諾再学習

ログなし利用

AI単独による無制限クロール

権限未定義の外部応答

投機性・金融商品性を持つ設計
